1. 建立工程基准：XGBoost

在动用神经网络这一“重型武器”之前，我们先用 XGBoost (梯度提升树) 建立基线。它训练极快，且对非线性关系有很好的捕捉能力。

优点

  • 训练速度极快 (秒级)
  • 不需要数据归一化
  • 可解释性强 (Feature Importance)

缺点

  • 频率外推能力差

“黑盒”模型的困境：

如果我们只使用 MSE Loss (均方误差) 训练：

$$ Loss = \frac{1}{N} \sum (S_{pred} - S_{true})^2 $$

结果往往是 S 参数曲线看起来重合得很好，但计算出来的 Q 值 (品质因数) 误差巨大。这是因为 Q 值对 S 参数的微小导数变化非常敏感。

$$ Q = \frac{\text{Im}(Z_{diff})}{\text{Re}(Z_{diff})} $$

3. 物理感知神经网络 (PINN)

核心思想：把电路公式写进 Loss Function

这是本项目的特色部分 (loss_functions.py)。我们不再只盯着 S 参数，而是要求模型预测出的 S 参数推导出的 $L$ 和 $Q$ 也必须准确。

# 代码片段：loss_functions.py / CombinedLoss
class CombinedLoss(nn.Module):
    def forward(self, S_pred, S_true, freq):
        # 1. 基础 S 参数误差
        loss_s = mse_loss(S_pred, S_true)
        
        # 2. 物理量计算 (在计算图中，可微分！)
        # 自动调用 KPICalculator 将 S -> Z -> L, Q
        kpi_pred = self.kpi_calculator(S_pred, freq)
        kpi_true = self.kpi_calculator(S_true, freq)
        
        # 3. 关键性能指标 (KPI) 误差
        loss_L = mse_loss(kpi_pred['L'], kpi_true['L'])
        loss_Q = mse_loss(kpi_pred['Q'], kpi_true['Q'])
        
        # 4. 总损失：物理感知
        return loss_s + w_kpi * (loss_L + loss_Q)

4. 自动化调优：Optuna

模型有多少层？学习率设多少？$w_{kpi}$ 设为 0.1 还是 1.0？手动尝试效率太低。

我们使用 Optuna 框架进行超参数搜索 (tune_nn.py)。它就像 ADS 中的 Optimizer，自动寻找 Loss 最低的参数组合。

搜索空间

  • lr: 1e-4 ~ 1e-2 (对数分布)
  • batch_size: 128, 256, 512
  • w_kpi: 0.0 ~ 2.0
  • hidden_layers: [64, 64] ~ [512, 512, 512]

优化目标

最大化综合评分：

$$ \text{Objective} = \alpha \cdot \text{MSE}_S + \beta \cdot \text{MAPE}_L + \gamma \cdot \text{MAPE}_Q $$

Optuna 会自动平衡 S 参数准确度和物理量准确度。
